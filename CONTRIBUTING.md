# 贡献指南

感谢你对 OpenClaw 多 Agent 权限治理方案的兴趣！我们欢迎各种形式的贡献。

---

## 如何贡献

### 报告问题

如果你发现文档中的错误或有改进建议：

1. 先搜索现有 Issues，避免重复
2. 创建新 Issue，使用以下模板：

```markdown
**问题类型**: [文档错误/改进建议/安全问题/其他]
**位置**: [文件路径和章节]
**当前内容**: [当前的问题描述]
**建议修改**: [你的改进建议]
**原因**: [为什么需要这个修改]
```

### 提交改进

#### 小修改（直接 PR）

- 拼写/语法修正
- 格式调整
- 链接修复

#### 大修改（先开 Issue 讨论）

- 新增文档章节
- 架构设计变更
- 新增模板类型

### PR 流程

1. Fork 本仓库
2. 创建分支: `git checkout -b feature/your-feature`
3. 提交修改: `git commit -m "描述你的修改"`
4. 推送分支: `git push origin feature/your-feature`
5. 创建 Pull Request

---

## 文档规范

### 格式要求

- 使用 Markdown 格式
- 文件名使用大写驼峰或 kebab-case
- 标题层级正确（H1 只能有一个）

### 内容要求

- 使用简体中文
- 技术术语保持英文（如 API、Cron、Agent）
- 提供具体示例
- 保持客观中立

## 行为准则

- 专注于建设性讨论
- 接受不同观点
