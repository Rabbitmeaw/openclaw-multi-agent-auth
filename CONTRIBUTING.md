# Contributing Guide

Thank you for your interest in the OpenClaw Multi-Agent Auth solution! We welcome contributions in all forms.

---

## How to Contribute

### Reporting Issues

If you find errors in documentation or have improvement suggestions:

1. First search existing Issues to avoid duplicates
2. Create a new Issue using the following template:

```markdown
**Issue Type**: [Documentation Error/Improvement Suggestion/Security Issue/Other]
**Location**: [File path and section]
**Current Content**: [Current problem description]
**Suggested Change**: [Your improvement suggestion]
**Reason**: [Why this change is needed]
```

### Submitting Improvements

#### Small Changes (Direct PR)

- Spelling/grammar corrections
- Format adjustments
- Link fixes

#### Large Changes (Discuss via Issue First)

- New documentation sections
- Architecture design changes
- New template types

### PR Process

1. Fork this repository
2. Create branch: `git checkout -b feature/your-feature`
3. Commit changes: `git commit -m "Describe your changes"`
4. Push branch: `git push origin feature/your-feature`
5. Create Pull Request

---

## Documentation Standards

### Format Requirements

- Use Markdown format
- Filenames use UpperCamelCase or kebab-case
- Correct heading levels (only one H1)

### Content Requirements

- Use English
- Keep technical terms in English (e.g., API, Cron, Agent)
- Provide concrete examples
- Maintain objectivity

---

## Code of Conduct

- Be respectful and kind to others
- Focus on constructive discussion
- Accept different viewpoints
